================================================================================
KEY PATTERNS & CONVENTIONS
================================================================================

1. CONTROLLER LAYER (Dual Tree)
   - Web Controllers:  app/Http/Controllers/[Entity]/[Entity]Controller.php
   - API Controllers:    app/Http/Controllers/Api/[Entity]Controller.php
   - Both share the same domain logic but different response formats

2. TRANSFORMER PATTERN
   - All API responses go through Transformer classes
   - Location: app/Http/Transformers/
   - Never return raw model attributes from API controllers

3. POLICY-BASED AUTHORIZATION
   - All authorization via Policies: app/Policies/
   - CheckoutablePermissionsPolicy for checkout/checkin operations

4. FMCS (Full Multiple Company Support)
   - Multi-tenant filtering via Setting::getSettings()->full_multiple_companies_support

5. SELECT2 AJAX DROPDOWNS
   - Auto-initialized via snipeit.js
   - Uses selectlist() API endpoints with companyId param

6. ACTION CLASS PATTERN
   - Single-responsibility business logic in app/Actions/[Entity]/
   - Replaces fat controller methods

7. ELOQUENT MODEL TRAITS
   - CompanyableScope: multi-company scoping
   - Checkoutable: checkout/checkin capability
   - Depreciable: depreciation calculation
